# BeKind | 互联网沟通指北

> **A Internet Communication Compass with Zen-iOS Design**  
> 一份现代互联网生存与沟通的元规则指南，采用极致的 Zen-iOS 设计风格。

![License](https://img.shields.io/github/license/DomeenoH/BeKind)
![Status](https://img.shields.io/badge/Status-Active-success)

**BeKind** 是一个旨在提升互联网沟通质量的开源项目。它收集并解释了一系列经典的“互联网定律”和“沟通剃刀”，帮助人们在网络交流中保持理性、高效和善意。

整个站点采用了 **Zen-iOS** 设计语言：磨砂玻璃质感、物理触觉反馈、以及清晰优雅的排版，旨在为阅读这些规则提供最舒适的体验。

## ✨ 特性 (Features)

*   **Zen-iOS 设计系统**：深度定制的 CSS 变量系统，实现了苹果风格的磨砂玻璃（Frosted Glass）、动态阴影和触觉反馈。
*   **沉浸式阅读体验**：针对长文本优化的排版，特殊引用块（Slack 模拟消息）经过精心对齐（`&nbsp;&nbsp;` 优化）。
*   **一键分享**：所有子页面的 Hero 区域集成“一键分享”功能，自动生成友好的分享文案 `这个或许对你有帮助 → [标题]：[链接]`，并带有“已复制”的视觉反馈。
*   **响应式布局**：完美支持移动端和桌面端访问。

## 🧭 收录法则 (The Laws)

本项目收录了 20+ 条核心互联网沟通法则，包括但不限于：

*   **小黄鸭调试法 (Rubber Duck Debugging)**：向鸭子解释问题往往能自己找到答案。
*   **汉隆剃刀 (Hanlon's Razor)**：能解释为愚蠢的，就不要解释为恶意。
*   **斯特金定律 (Sturgeon's Law)**：90% 的东西都是垃圾。
*   **提问的智慧 (Smart Questions)**：如何提出一个容易得到解答的好问题。
*   **XY 问题 (The XY Problem)**：不要问你认为的解决方案，直接问你的问题。
*   **不要说"在吗" (No Hello)**：直接说事，尊重他人的时间。
*   ... 以及更多。

## 🛠️ 开发 (Development)

本项目基于 [Vite](https://vitejs.dev/) 构建，使用原生 HTML/CSS/JS 开发，保持轻量级和高性能。

### 环境要求
*   Node.js 16+
*   npm

### 启动项目
```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

### 构建部署
```bash
# 构建生产版本
npm run build
```
